ABOUT -- EXTREME ASSAULT MUSIC / GENERAL MIDI / MT-32 / ADLIB

The cockpit score, three ways.

Extreme Assault (Blue Byte Software, 1997) keeps its in-mission soundtrack as Red Book CD audio, but the menus, briefings and FM build of the score live on disk as compact Miles Sound System XMI event data in DATA.SFX/MUSIC — not as recorded audio. This jukebox plays that sequenced data three ways: the General MIDI mix (for a Roland SC-55 / SCC-1 Sound Canvas), a Roland MT-32 rendition, and the game's own AdLib FM build. Use the Gen MIDI / MT-32 / AdLib filter to switch between them. (The streamed CD tracks are a different recording and are not included here.)

General MIDI: the Sound Canvas mix

The General-MIDI arrangement targets a Roland SC-55 / SCC-1, the Sound Canvas that defined the GM sound of the mid-90s. It is fully multi-timbral, one part per MIDI channel with standard GM instruments and a dedicated drum channel, so it sounds cleaner and fuller than the FM build while staying faithful to the original cues.

Roland MT-32: the stock passthrough driver

Extreme Assault ships a genuine MT-32 driver, MT32MPU.MDI (“Roland MT-32 MIDI with MPU-401 MIDI Interface”, also covering the LAPC-I). Unlike the fan driver used for the Warcraft II jukebox, this is the stock Miles MPU-401 output driver: a plain passthrough that streams the cue's MIDI straight to the synth with no GM→MT-32 translation. So this rendition is the exact same note/program data as the General-MIDI mix, played on Roland's MT-32 / CM-32L (emulated by Munt) instead of the Sound Canvas. The MT-32's preset map is its own — not GM-ordered — so the instruments differ from the SC-55 version; that is precisely what an MT-32 owner heard with this driver.

AdLib: the game's own FM bank

The AdLib build is the game's own FM synthesis. Extreme Assault's Miles drivers map each General-MIDI program through a custom OPL instrument bank shipped as LEVEL_X.OPL (a Miles/AIL timbre library: 128 melodic + percussion 2-operator patches). We convert that bank to WOPL, feed the original XMI through a patched build of libADLMIDI loaded with it, capture every OPL register write, and replay it through a cycle-accurate Nuked-OPL3 emulator — so the timbres, voice allocation and key-scaling are the card's own, not a generic AdLib substitute.

How this works

Each cue in DATA.SFX/MUSIC is an AIL/Miles XMI (XMIDI) stream. The General-MIDI XMI is converted to a Standard MIDI File (a port of Exult's XMidiFile timing model); the MT-32 path plays that same SMF through Munt; the FM XMI is rendered by libADLMIDI with the game's LEVEL_X bank, captured as a DOSBox raw-OPL (DRO) register log. All three are then played in your browser:

1997 Extreme Assault DATA.SFX/MUSIC (XMI)-> SMF / OPL register log (game LEVEL_X bank)-> SMF / DRO-> JavaScript timing-> WebAssembly Nuked-SC55 (GM) / Munt CM-32L (MT-32) / Nuked-OPL3 (AdLib)-> synthesized sound-> modern headphones

The synthesis engine, ROM images and UI are shared with the sister TFX jukebox -- only the music converters and this page are new.
